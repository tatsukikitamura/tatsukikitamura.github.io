# デザインリデザイン仕様書

## 概要

ポートフォリオサイト（tatsuki.dev）のデザインを全面的に刷新する。

## 現状の課題

- グラデーション・orb・タイピングアニメーションなど演出過多
- 絵文字を多用している（コンテンツ中・アイコン代替として使用）
- 派手なアニメーションが「安心感」より「派手さ」を演出している
- LinkedInのような「信頼・実績を伝える」デザインになっていない

---

## 目指すデザイン方向性

### コンセプト
**「シンプルで見やすく、安心感のあるプロフェッショナルなポートフォリオ」**

LinkedInのような、経験・実績・スキルが一目でわかる清潔感のあるレイアウト。

### 参考: LinkedInの特徴
- 白背景 + ライトグレーのカード
- セクションが明確に区切られている
- テキスト中心、余白が適切
- 信頼感・安心感のある青系アクセントカラー
- アイコンは公式ロゴ・SVGを使用（絵文字NG）

---

## デザイン仕様（確定済み）

### 禁止事項
- 絵文字の使用（コンテンツ中・アイコン代替すべて禁止）
- 大きなグラデーション背景・orbアニメーション
- 過剰な演出アニメーション

### 許可事項
- 公式ロゴ・ブランドアイコン（GitHub, Python, Rails 等の SVG ロゴ）
- 控えめなフェードイン等のアニメーション（過剰でなければOK）
- タイピングアニメーションは控えめに縮小して残す

### カラーパレット
| 用途 | カラーコード |
|------|-------------|
| 背景（メイン） | `#ffffff` |
| 背景（セカンダリ） | `#f3f4f6` |
| テキスト（メイン） | `#111827` |
| テキスト（サブ） | `#6b7280` |
| アクセント | `#0a66c2`（LinkedIn blue） |
| カード背景 | `#ffffff` |
| カードボーダー | `#e5e7eb` |

### タイポグラフィ
- ボディ: **Inter**（Google Fonts）
- コード: JetBrains Mono（現行維持）
- 日本語補完: システムフォント fallback（または Noto Sans JP）

### レイアウト
- 最大幅: `max-w-3xl` または `max-w-4xl` の中央寄せ
- カード: 角丸 `rounded-xl`、白背景、薄ボーダー
- Experience ページ: タイムライン形式（左縦線 + ドット + カード）

### ヘッダー
- **常に白固定**（スクロール位置に関わらず白背景を維持）

### ナビゲーション
- 現行: Projects / Experience / Blog / Contact
- 現行維持の方針（Blogは技術的差別化要素として残す）
- 将来的に Skills ページや About ページの追加も検討可

### アニメーション方針
- フェードイン等の控えめなアニメーションは維持
- 過剰な演出（orb、グラデーションシフト、タイピング全文）は削除・縮小

### 言語
- 日英混在のまま維持

## ページ別方針

### トップページ (index.html / src/main.js)
- タイピングアニメーション: 短い1文のみに縮小（現行の長いコードブロックは削除）
- Hero: 名前 + 肩書き + 一言説明 + SNSリンクアイコン（GitHub等）
- 顔写真: 使用しない
- Featured Work セクション: カードデザインを刷新（絵文字 → 公式ロゴ）
- orb・グラデーション背景: 削除

### Experience ページ (src/pages/experience.js)
- 絵文字アイコン（現行）→ シンプルなSVGアイコンまたは会社/カテゴリロゴに変更
- LinkedInスタイルのタイムライン: 左縦線 + 各エントリーをカードで表示

### Projects ページ (src/pages/projects.js 他)
- 絵文字 → 技術スタックの公式ロゴに変更
- カードデザインを白 + ライトボーダーに統一

### 共通コンポーネント
- Header (src/components/header.js): 確認中（スクロール挙動の方針待ち）
- Footer (src/components/footer.js): 最小限のシンプルな構成を維持
